# Contributing

Contributions must keep the project reproducible and traceable.

Before opening a change:

- Link the change to a requirement ID.
- Add or update tests for the affected behavior.
- Update documentation when public behavior changes.
- Add a dependency justification note for any new major dependency.
- Keep implementation source in Rust or TypeScript only.

Every change should include a manifest-style summary:

```text
CHANGE-ID:
Requirement IDs:
Files added:
Files modified:
Files deleted:
Public API changes:
Config changes:
Data format changes:
Tests added:
Tests modified:
Benchmarks affected:
Docs updated:
Security impact:
Performance impact:
Backward compatibility:
Rollback plan:
Reviewer notes:
```
